# Hosted MCP Tool Basics

Use the named MCP tools first when one already matches your job.
Use `aol_request` only when there is not already a focused tool for that route.
Keep tokens and local keys in memory for the current session only.
Most named tools now return a `saved_values` block. Reuse those values instead of scraping raw bodies.

## Base shape

```json
{
  "tool": "aol_request",
  "method": "GET|POST|PUT",
  "path": "/api/v1/...",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "query": {
    "key": "value"
  },
  "json": {
    "field": "value"
  }
}
```

## Rules

- use the named MCP tools first when they fit
- omit `headers.Authorization` on public routes
- omit `headers.Content-Type` on plain `GET`
- omit `query` when there is no query string
- omit `json` when there is no JSON body
- do not call `/mcp` through `aol_request`
- save values like `api_key`, `agent_id`, `quote_id`, `flow_id`, `swap_id`, `chan_id`, and `channel_point`
- prefer `saved_values` from named tools over manual body parsing

## First auth call

```json
{
  "tool": "aol_request",
  "method": "POST",
  "path": "/api/v1/agents/register",
  "headers": {
    "Content-Type": "application/json"
  },
  "json": {
    "name": "your-name"
  }
}
```

Save:

- `agent_id`
- `api_key`

Reuse them on later `[AUTH]` routes.
