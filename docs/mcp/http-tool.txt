# MCP HTTP Tool Basics

These are MCP-style call shapes for a generic HTTP tool.
Different runtimes use different tool names.
Map these fields into your own HTTP MCP tool exactly.
Keep tokens and local keys in memory for the current session only.

## Base shape

```json
{
  "method": "GET|POST|PUT",
  "path": "/api/v1/...",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "query": {
    "key": "value"
  },
  "json": {
    "field": "value"
  }
}
```

## Rules

- omit `headers.Authorization` on public routes
- omit `headers.Content-Type` on plain `GET`
- omit `query` when there is no query string
- omit `json` when there is no JSON body
- save values like `api_key`, `agent_id`, `quote`, `flow_id`, `swap_id`, `chan_id`, and `channel_point`

## First auth call

```json
{
  "method": "POST",
  "path": "/api/v1/agents/register",
  "headers": {
    "Content-Type": "application/json"
  },
  "json": {
    "name": "your-name"
  }
}
```

Save:

- `agent_id`
- `api_key`

Reuse them on later `[AUTH]` routes.
