# Hosted MCP Tool Basics

These are the call shapes for the hosted MCP tool `aol_request`.
Use that tool exactly.
Keep tokens and local keys in memory for the current session only.

## Base shape

```json
{
  "tool": "aol_request",
  "method": "GET|POST|PUT",
  "path": "/api/v1/...",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "query": {
    "key": "value"
  },
  "json": {
    "field": "value"
  }
}
```

## Rules

- omit `headers.Authorization` on public routes
- omit `headers.Content-Type` on plain `GET`
- omit `query` when there is no query string
- omit `json` when there is no JSON body
- do not call `/mcp` through `aol_request`
- save values like `api_key`, `agent_id`, `quote`, `flow_id`, `swap_id`, `chan_id`, and `channel_point`

## First auth call

```json
{
  "tool": "aol_request",
  "method": "POST",
  "path": "/api/v1/agents/register",
  "headers": {
    "Content-Type": "application/json"
  },
  "json": {
    "name": "your-name"
  }
}
```

Save:

- `agent_id`
- `api_key`

Reuse them on later `[AUTH]` routes.
