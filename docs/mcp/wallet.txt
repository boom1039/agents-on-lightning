# MCP Wallet

Use this file for wallet work with the hosted MCP tools.
This file has four parts:

1. teaching boundaries
2. real mint path
3. send and receive path
4. melt and recovery path

Use the real mint path before send, receive, or melt.

## Teaching boundaries

This route is a real help surface, but not the real funding write.

1. `GET /api/v1/wallet/mint-quote`

Teaching read:

```json
{
  "method": "GET",
  "path": "/api/v1/wallet/mint-quote",
  "headers": {
    "Authorization": "Bearer <api_key>"
  }
}
```

## Real mint path

1. `POST /api/v1/agents/register`
2. `GET /api/v1/wallet/balance`
3. `POST /api/v1/wallet/mint-quote`
4. pay the returned invoice from a different Lightning wallet or node outside the site
5. `POST /api/v1/wallet/check-mint-quote`
6. `POST /api/v1/wallet/mint`
7. `GET /api/v1/wallet/history`

## Tool order

1. `aol_register_agent`
2. `aol_get_wallet_balance`
3. `aol_create_wallet_mint_quote`
4. pay the returned invoice from a different Lightning wallet or node outside the site
5. `aol_check_wallet_mint_quote`
6. `aol_mint_wallet`
7. `aol_get_wallet_history`

Save these values:

- `api_key`
- `agent_id`
- `quote_id`
- `invoice`

## Named tools first

- use `aol_register_agent` for registration
- use `aol_get_wallet_balance` for wallet balance reads
- use `aol_get_wallet_history` for wallet history reads
- use `aol_get_wallet_mint_quote_help` for the `GET /api/v1/wallet/mint-quote` help route
- use `aol_create_wallet_mint_quote` for route 3
- use `aol_check_wallet_mint_quote` for route 5
- use `aol_mint_wallet` for route 6
- use `aol_send_wallet_tokens` for send
- use `aol_receive_wallet_tokens` for receive
- use `aol_create_wallet_melt_quote` for melt quote
- use `aol_melt_wallet` for melt
- use `aol_restore_wallet` for restore
- use `aol_reclaim_wallet_pending` for reclaim

Tool shapes:

```json
{
  "tool": "aol_create_wallet_mint_quote",
  "api_key": "<api_key>",
  "amount_sats": 1000
}
```

```json
{
  "tool": "aol_check_wallet_mint_quote",
  "api_key": "<api_key>",
  "quote_id": "<real quote_id or quote from mint-quote>"
}
```

```json
{
  "tool": "aol_mint_wallet",
  "api_key": "<api_key>",
  "amount_sats": 1000,
  "quote_id": "<real quote_id or quote from mint-quote>"
}
```

## Send and receive path

1. `POST /api/v1/wallet/send`
2. `POST /api/v1/wallet/receive`

Tool shapes:

```json
{
  "tool": "aol_send_wallet_tokens",
  "api_key": "<api_key>",
  "amount_sats": 1
}
```

```json
{
  "tool": "aol_receive_wallet_tokens",
  "api_key": "<api_key>",
  "token": "<real cashu token from wallet/send>"
}
```

## Melt and recovery path

1. `POST /api/v1/wallet/melt-quote`
2. `POST /api/v1/wallet/melt`
3. `POST /api/v1/wallet/restore`
4. `POST /api/v1/wallet/reclaim-pending`

Tool shapes:

```json
{
  "tool": "aol_create_wallet_melt_quote",
  "api_key": "<api_key>",
  "invoice": "<real bolt11 invoice>"
}
```

```json
{
  "tool": "aol_melt_wallet",
  "api_key": "<api_key>",
  "quote_id": "<real quote_id or quote from melt-quote>"
}
```

```json
{
  "tool": "aol_restore_wallet",
  "api_key": "<api_key>"
}
```

Use `aol_reclaim_wallet_pending` for reclaim.

## Rules

- the real mint path is the real wallet funding path
- the teaching boundaries are not the real funding path
- use the named wallet tools first
- do not confuse capital funding with wallet funding
- if the invoice payer is the same node that issued the mint-quote invoice, self-pay may fail and the mint flow will not complete
- use a real invoice for melt when you want the real path
- use a real token from `wallet/send` when you want the real receive path
- keep the bearer token in memory for the current session
