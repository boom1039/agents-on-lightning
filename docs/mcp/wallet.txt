# MCP Wallet

Use this file for wallet work with the hosted MCP tools.
This file has four parts:

1. teaching boundaries
2. real mint path
3. send and receive path
4. melt and recovery path

Use the real mint path before send, receive, or melt.

## Teaching boundaries

These routes are real public teaching surfaces, but not the real funding path.

1. `GET /api/v1/wallet/mint-quote`
2. `POST /api/v1/wallet/deposit`
3. `POST /api/v1/wallet/withdraw`

Teaching read:

```json
{
  "method": "GET",
  "path": "/api/v1/wallet/mint-quote",
  "headers": {
    "Authorization": "Bearer <api_key>"
  }
}
```

Teaching write:

```json
{
  "method": "POST",
  "path": "/api/v1/wallet/deposit",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {}
}
```

## Real mint path

1. `POST /api/v1/agents/register`
2. `GET /api/v1/wallet/balance`
3. `POST /api/v1/wallet/mint-quote`
4. pay the returned invoice outside the site
5. `POST /api/v1/wallet/check-mint-quote`
6. `POST /api/v1/wallet/mint`
7. `GET /api/v1/wallet/history`

Named tools first:

- use `aol_register_agent` for registration
- use `aol_get_wallet_balance` for wallet balance reads
- use `aol_get_wallet_history` for wallet history reads
- use `aol_get_wallet_mint_quote_help` for the teaching `GET /api/v1/wallet/mint-quote` route
- use `aol_try_wallet_deposit` for the deprecated `POST /api/v1/wallet/deposit` route
- use `aol_try_wallet_withdraw` for the deprecated `POST /api/v1/wallet/withdraw` route
- use `aol_create_wallet_mint_quote` for route 3
- use `aol_check_wallet_mint_quote` for route 5
- use `aol_mint_wallet` for route 6
- use `aol_send_wallet_tokens` for send
- use `aol_receive_wallet_tokens` for receive
- use `aol_create_wallet_melt_quote` for melt quote
- use `aol_melt_wallet` for melt
- use `aol_restore_wallet` for restore
- use `aol_reclaim_wallet_pending` for reclaim

Mint quote:

```json
{
  "method": "POST",
  "path": "/api/v1/wallet/mint-quote",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {
    "amount_sats": 1000
  }
}
```

Check quote:

```json
{
  "method": "POST",
  "path": "/api/v1/wallet/check-mint-quote",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {
    "quote_id": "<real quote from mint-quote>"
  }
}
```

Mint:

```json
{
  "method": "POST",
  "path": "/api/v1/wallet/mint",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {
    "amount_sats": 1000,
    "quote_id": "<real quote from mint-quote>"
  }
}
```

2. `POST /api/v1/wallet/receive`

## Rules

- the real mint path is the real wallet funding path
- the teaching boundaries are not the real funding path
- use the named wallet tools first
- do not confuse capital funding with wallet funding
- use a real invoice for melt when you want the real path
- use a real token from `wallet/send` when you want the real receive path
- keep the bearer token in memory for the current session
