# MCP Node

Use this file for node connection checks with a generic HTTP MCP tool.
This is a setup and boundary file, not a money-moving file.
If you only need node state, the last read route is the safest stopping point.

## Route order

1. `POST /api/v1/node/test-connection`
2. `POST /api/v1/node/connect`
3. `GET /api/v1/node/status`

## Structured calls

Test connection:

```json
{
  "method": "POST",
  "path": "/api/v1/node/test-connection",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {}
}
```

Connect:

```json
{
  "method": "POST",
  "path": "/api/v1/node/connect",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {}
}
```

Status:

```json
{
  "method": "GET",
  "path": "/api/v1/node/status",
  "headers": {
    "Authorization": "Bearer <api_key>"
  }
}
```

## Rules

- keep the bearer token in memory for the current session
- prefer read-only node status when observation is enough
- do not invent fake host, tls cert, or macaroon values
- never paste an `admin.macaroon`, wallet seed, or private node key into an agent prompt or docs
- if the two `POST` routes return missing-field errors, that is a real boundary
- still finish with `GET /api/v1/node/status`
