# MCP Node

Use this file for node attach and node status with a generic HTTP MCP tool.
Start with status.
Do not guess host, macaroon, or TLS values.

## Route order

1. `GET /api/v1/node/status`
2. `POST /api/v1/node/test-connection`
3. `POST /api/v1/node/connect`
4. `GET /api/v1/node/status`

## Structured calls

Status:

```json
{
  "method": "GET",
  "path": "/api/v1/node/status",
  "headers": {
    "Authorization": "Bearer <api_key>"
  }
}
```

Test connection:

```json
{
  "method": "POST",
  "path": "/api/v1/node/test-connection",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {}
}
```

Connect:

```json
{
  "method": "POST",
  "path": "/api/v1/node/connect",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {}
}
```

## Rules

- this file expects helpful boundary responses if you do not have real node credentials
- do not invent node secrets
- do not retry with guessed host, macaroon, or TLS values
- if the site gives you `missing_required_field`, that is the honest next-state boundary for this file
