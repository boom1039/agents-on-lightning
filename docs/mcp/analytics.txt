# MCP Analytics

Use this file for analytics quotes, execution, and history with the hosted MCP tools.
This is an auth flow.

## Route order

1. `POST /api/v1/agents/register`
2. `GET /api/v1/analytics/catalog`
3. `POST /api/v1/analytics/quote`
4. `POST /api/v1/analytics/execute`
5. `GET /api/v1/analytics/history`

Named tools first:

- use `aol_register_agent` for route 1
- use `aol_get_analytics_catalog` for route 2
- use `aol_quote_analytics` for route 3
- use `aol_execute_analytics` for route 4
- use `aol_get_analytics_history` for route 5

Catalog:

```json
{
  "method": "GET",
  "path": "/api/v1/analytics/catalog"
}
```

Quote:

```json
{
  "method": "POST",
  "path": "/api/v1/analytics/quote",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {
    "query_id": "network_stats",
    "params": {}
  }
}
```

Execute:

```json
{
  "method": "POST",
  "path": "/api/v1/analytics/execute",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {
    "query_id": "network_stats",
    "params": {}
  }
}
```

History:

```json
{
  "method": "GET",
  "path": "/api/v1/analytics/history",
  "headers": {
    "Authorization": "Bearer <api_key>"
  }
}
```

## Rules

- read the catalog before quote
- ask for a quote before execute
- execute will use wallet ecash first and available capital second if wallet balance is too low
- if execute returns a paid boundary or funding error, still read history
- keep the bearer token in memory for this session
