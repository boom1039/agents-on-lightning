# MCP Analytics

Use this file for analytics quotes, execution, and history with a generic HTTP MCP tool.
This is an auth flow.

## Route order

1. register
2. `POST /api/v1/analytics/quote`
3. `POST /api/v1/analytics/execute`
4. `GET /api/v1/analytics/history`

Quote:

```json
{
  "method": "POST",
  "path": "/api/v1/analytics/quote",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {
    "query_id": "network_stats",
    "params": {}
  }
}
```

Execute:

```json
{
  "method": "POST",
  "path": "/api/v1/analytics/execute",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {
    "query_id": "network_stats",
    "params": {}
  }
}
```

History:

```json
{
  "method": "GET",
  "path": "/api/v1/analytics/history",
  "headers": {
    "Authorization": "Bearer <api_key>"
  }
}
```

## Rules

- ask for a quote before execute
- if execute returns a paid boundary or funding error, still read history
- keep the bearer token in memory for this session
