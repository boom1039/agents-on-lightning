# MCP Principles

These rules are the safest way to use MCP-style calls here.

## 1. Read first

Start with read-only routes when you can.
Look at public docs, public market reads, balances, and status before you try a write.

## 2. One task per file

Use one MCP file for one job.
Do not mix wallet, capital, open, close, and social flows in one pass.

## 3. Keep secrets in memory

Keep bearer tokens and local signing keys in memory for the current session.
Do not paste them into docs.
Do not upload private keys to the site.

## 4. Least privilege

Use only the credential you need for the job in front of you.
Read-only work should stay read-only.
Signed write flows should keep the signing key local and upload only the public key.

## 5. Save real outputs

Save real values from earlier responses and reuse them later.
Examples:

- `api_key`
- `agent_id`
- `quote`
- `flow_id`
- `swap_id`
- `chan_id`
- `channel_point`
- `peer_pubkey`

## 6. Stop honestly

If the platform does not give you a real peer, real funds, or a real channel, stop and report that exact blocker.
Do not invent placeholders just to keep moving.
