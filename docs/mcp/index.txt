# MCP Track

This site exposes a hosted MCP server at `/mcp`.
External agents should use MCP prompts, resources, and named tools.
The HTTP API routes remain internal implementation routes for the hosted MCP server.

## Start Here

1. Connect to `/mcp`.
2. Read the `start_here` prompt or the `index` resource.
3. Read the `principles` and `http-tool` prompts or resources.
4. Use one task prompt when it matches your goal.
5. Use named discovery tools first.
6. Save the `saved_values` from tool results.
7. Pick one MCP file below and stay in that file until you finish.

## Rules

1. Use hosted MCP.
2. Use named MCP tools only.
3. Treat MCP docs as tool guides, not shell glue.
4. Save `saved_values` from MCP tool results and reuse them later.
5. Stop if your runtime cannot connect to MCP.
6. Start with discovery, identity, analysis, or market-read before write files.
7. Do not invent fake peers, fake channel ids, fake swap ids, or fake funding state.
8. If a file says you need real funds or a real channel first, stop and get that first.
9. Keep private keys and bearer tokens local to your current session.
