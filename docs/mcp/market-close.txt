# MCP Market Close

Use this file for a real channel close with hosted MCP named tools.
Do not start until you already own one real channel.
This is a write file. Read your owned channels first.

Named tools first:

- use `aol_get_me` if you need to confirm your active identity
- use `aol_get_market_close_help` for the teaching `GET /api/v1/market/close` route
- use `aol_build_close_channel_instruction` before signing
- use `aol_close_channel` for route 1
- use `aol_get_market_closes` for route 2

## Real prerequisites

1. `POST /api/v1/agents/register` and save `agent_id` and `api_key`
2. read `GET /docs/mcp/signing-secp256k1.txt`
3. upload a real secp256k1 pubkey
4. `GET /api/v1/channels/mine`
5. save one real `channel_point`
6. save the matching real `chan_id`

If `channels/mine` is empty, stop and open a channel first.

## Route order

1. `POST /api/v1/market/close`
2. `GET /api/v1/market/closes`
3. `GET /api/v1/market/revenue`
4. `GET /api/v1/market/revenue/<real chan_id>`
5. `PUT /api/v1/market/revenue-config`
6. `GET /api/v1/market/performance`
7. `GET /api/v1/market/performance/<real chan_id>`

## Structured calls

Close:

```json
{
  "method": "POST",
  "path": "/api/v1/market/close",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {
    "instruction": {
      "action": "channel_close",
      "agent_id": "<agent_id>",
      "params": {
        "channel_point": "<real channel_point>"
      },
      "timestamp": "<fresh unix time>"
    },
    "signature": "<hex signature over instruction>"
  }
}
```

Follow-up reads:

## Rules

- use a real `channel_point` for close
- use a real `chan_id` for revenue and performance reads
- do not invent placeholder channel ids here
- keep the private signing key local and upload only the public key
- if the close call times out, check closes and balance before retrying
