# MCP Market Liquidity

Use this file for swaps, ecash-funded channel opens, and rebalances.
This is a write file.
Do the read, wallet, capital, analysis, and signing work first.

## Real prerequisites

1. `POST /api/v1/agents/register` and save `agent_id` and `api_key`
2. upload a real secp256k1 pubkey
3. keep a local signing key
4. for swaps, have a real Bitcoin address
5. for fund-from-ecash, have real ecash in the wallet and one real peer pubkey
6. for rebalance, already own one real channel

## Route order

1. `GET /api/v1/market/swap/quote?amount_sats=100000`
2. `POST /api/v1/market/swap/lightning-to-onchain`
3. `GET /api/v1/market/swap/status/<real swap_id>`
4. `GET /api/v1/market/swap/history`
5. `POST /api/v1/market/fund-from-ecash`
6. `GET /api/v1/market/fund-from-ecash/<real flow_id>`
7. `POST /api/v1/market/rebalance/estimate`
8. `POST /api/v1/market/rebalance`
9. `GET /api/v1/market/rebalances`

## Structured calls

Swap create:

```json
{
  "method": "POST",
  "path": "/api/v1/market/swap/lightning-to-onchain",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {
    "amount_sats": 100000,
    "onchain_address": "<real bitcoin address>"
  }
}
```

Fund from ecash:

```json
{
  "method": "POST",
  "path": "/api/v1/market/fund-from-ecash",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {
    "instruction": {
      "action": "channel_open",
      "agent_id": "<agent_id>",
      "params": {
        "local_funding_amount_sats": 100000,
        "peer_pubkey": "<real peer pubkey>"
      },
      "timestamp": "<fresh unix time>"
    },
    "signature": "<hex signature over instruction>"
  }
}
```

Rebalance estimate:

```json
{
  "method": "POST",
  "path": "/api/v1/market/rebalance/estimate",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {
    "outbound_chan_id": "<real owned chan_id>",
    "amount_sats": 10000
  }
}
```

Rebalance:

```json
{
  "method": "POST",
  "path": "/api/v1/market/rebalance",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {
    "instruction": {
      "action": "rebalance",
      "agent_id": "<agent_id>",
      "params": {
        "amount_sats": 10000,
        "max_fee_sats": 10,
        "outbound_chan_id": "<real owned chan_id>"
      },
      "timestamp": "<fresh unix time>"
    },
    "signature": "<hex signature over instruction>"
  }
}
```

## Rules

- do not invent a fake swap id, flow id, peer pubkey, or chan_id
- if swap create does not return a real `swap_id`, stop before route 3 and still do route 4
- if fund-from-ecash does not return a real `flow_id`, stop before route 6
- if you do not own a real channel, stop before routes 7 and 8
- sign only the inner `instruction` object
