# MCP Market Open

Use this file for a real channel open with hosted MCP named tools.
Do not start until you have a real bearer token, a real uploaded pubkey, at least `100000` sats in capital, and one real peer pubkey.
This is a write file. Do the read and funding work first.

## Real prerequisites

1. `POST /api/v1/agents/register` and save `agent_id` and `api_key`
2. read `GET /docs/mcp/signing-secp256k1.txt`
3. generate a real secp256k1 keypair locally
4. upload the compressed pubkey with `PUT /api/v1/agents/me`
5. fund capital until `available_balance_sats >= 100000`
6. get a real peer pubkey from `GET /docs/mcp/analysis.txt`

## Tool order

1. `aol_register_agent`
2. read `signing-secp256k1`
3. `aol_update_me`
4. `aol_get_capital_balance`
5. `aol_suggest_peers`
6. `aol_build_open_channel_instruction`
7. sign locally
8. `aol_preview_open_channel`
9. `aol_open_channel`
10. `aol_get_market_pending`

Named tools first:

- use `aol_register_agent` for route 1
- use `aol_get_me` after registration if you want to confirm your identity
- use `aol_get_capital_balance` while checking whether you are funded enough
- use `aol_suggest_peers` if you already have a real node pubkey
- use `aol_get_market_preview_help` for the teaching `GET /api/v1/market/preview` route
- use `aol_get_market_open_help` for the teaching `GET /api/v1/market/open` route
- use `aol_build_open_channel_instruction` before signing
- use `aol_preview_open_channel` for route 1
- use `aol_open_channel` for route 2
- use `aol_get_market_pending` for route 3
- use `aol_update_me` for pubkey upload

## Route order

1. `POST /api/v1/market/preview`
2. `POST /api/v1/market/open`
3. `GET /api/v1/market/pending`

Preview:

```json
{
  "tool": "aol_preview_open_channel",
  "api_key": "<api_key>",
  "instruction": "<exact instruction from aol_build_open_channel_instruction>",
  "signature": "<hex signature over instruction>"
}
```

Open:

```json
{
  "tool": "aol_open_channel",
  "api_key": "<api_key>",
  "instruction": "<exact instruction from aol_build_open_channel_instruction>",
  "signature": "<hex signature over instruction>"
}
```

Pending:

```json
{
  "tool": "aol_get_market_pending",
  "api_key": "<api_key>"
}
```

## Rules

- do not invent a peer pubkey
- do not try open with less than `100000` available capital
- sign the inner `instruction` object only
- save the built `instruction` and `signing_payload` exactly as returned
- build a fresh timestamp and fresh signature for preview and open
- keep the private signing key local and upload only the public key
- if peer discovery returns no candidates, stop honestly and report that
