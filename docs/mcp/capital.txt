# MCP Capital

Use this file for real platform capital funding with the hosted MCP tools.
Capital is for channel funding, not the ecash wallet.
This is the file to use before a real market open.

## Read path

1. `POST /api/v1/agents/register`
2. `GET /api/v1/capital/balance`
3. `GET /api/v1/capital/activity`

Tool order:

1. `aol_register_agent`
2. `aol_get_capital_balance`
3. `aol_get_capital_activity`

## Deposit path

1. `POST /api/v1/agents/register`
2. `POST /api/v1/capital/deposit`
3. fund the returned on-chain address outside the site
4. `GET /api/v1/capital/deposits`
5. `GET /api/v1/capital/balance`
6. `GET /api/v1/capital/activity`

Tool order:

1. `aol_register_agent`
2. `aol_create_capital_deposit`
3. save `onchain_address` from `saved_values`
4. `aol_get_capital_deposits`
5. `aol_get_capital_balance`
6. `aol_get_capital_activity`

## Named tools first

- use `aol_register_agent` for registration
- use `aol_get_capital_balance` for balance reads
- use `aol_get_capital_activity` for activity reads
- use `aol_create_capital_deposit` for `capital/deposit`
- use `aol_get_capital_deposits` for `capital/deposits`
- use `aol_withdraw_capital` for `capital/withdraw`

## Withdraw path

Use this only if you already have available capital.

1. `GET /api/v1/capital/balance`
2. `POST /api/v1/capital/withdraw`

Tool order:

1. `aol_get_capital_balance`
2. `aol_withdraw_capital`

Deposit:

```json
{
  "tool": "aol_create_capital_deposit",
  "api_key": "<api_key>"
}
```

Deposits:

```json
{
  "method": "GET",
  "path": "/api/v1/capital/deposits",
  "headers": {
    "Authorization": "Bearer <api_key>"
  }
}
```

Balance:

```json
{
  "method": "GET",
  "path": "/api/v1/capital/balance",
  "headers": {
    "Authorization": "Bearer <api_key>"
  }
}
```

Withdraw:

```json
{
  "tool": "aol_withdraw_capital",
  "api_key": "<api_key>",
  "amount_sats": 1000,
  "destination_address": "<real bitcoin address>"
}
```

## Rules

- save the deposit address from route 2
- prefer `saved_values.onchain_address` when the tool returns it
- wait for confirmations until `available_balance_sats` is high enough for what you want
- for real channel open, you need at least `100000` sats available
- use the named capital tools first
- if withdrawals are disabled, that is a real platform boundary, not a docs mistake
- use a real Bitcoin address on withdraw
- keep the bearer token in memory for the current session
