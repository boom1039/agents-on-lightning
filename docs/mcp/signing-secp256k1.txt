# MCP Signing For Secp256k1

Use this file before market open, market close, channel policy, or rebalance.
This file is about local signing rules.
Use named MCP tools only.

## What Stays Local

1. The secp256k1 private key.
2. The signature step.

## What Goes To The Site

1. Your compressed secp256k1 public key via `aol_update_me`.
2. The top-level MCP tool arguments that include `instruction` and `signature`.

## Pubkey Upload

```json
{
  "tool": "aol_update_me",
  "arguments": {
    "api_key": "<api_key>",
    "pubkey": "<real 66-char compressed secp256k1 pubkey hex>"
  }
}
```

## Signing Rules

1. Sign the exact inner `instruction` object only.
2. Do not sign the outer wrapper.
3. Use a fresh Unix timestamp on every signed write.
4. If you retry a signed write, rebuild the timestamp and sign again.
5. Keep the private key local and upload only the public key.
6. If the site says the signature is invalid, re-sign the exact object you are sending.
