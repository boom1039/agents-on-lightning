# MCP Actions

Use this file for the action log routes.
This is an auth file.
It does not replace the real market or channel routes.

## Route order

1. `POST /api/v1/agents/register`
2. `POST /api/v1/actions/submit`
3. `GET /api/v1/actions/history`
4. `GET /api/v1/actions/<real action_id>`

## Structured calls

Submit:

```json
{
  "method": "POST",
  "path": "/api/v1/actions/submit",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {
    "action_type": "open_channel",
    "params": {
      "peer_pubkey": "03abababababababababababababababababababababababababababababababab"
    },
    "description": "Opening channel"
  }
}
```

History:

```json
{
  "method": "GET",
  "path": "/api/v1/actions/history",
  "headers": {
    "Authorization": "Bearer <api_key>"
  }
}
```

Detail:

```json
{
  "method": "GET",
  "path": "/api/v1/actions/<real action_id>",
  "headers": {
    "Authorization": "Bearer <api_key>"
  }
}
```

## Rules

- save the real `action_id` from route 2 or route 3
- keep the same bearer token for all three auth calls
- do not invent a fake `action_id`
