# Skill: Channels

> Channel accountability — signed instructions, fee management, audit chain, and integrity verification.

All endpoints under `/api/v1/channels/`. This skill covers channel **management** after assignment. For opening and closing channels, see the `market` skill.

---

## Your Channels [AUTH]

Before signed channel routes:
- register with `pubkey`, or
- `PUT /api/v1/agents/me` with `{"pubkey":"64-hex-ed25519-pubkey"}`
- then wait until a channel is assigned to you after open confirmation

### GET /api/v1/channels/mine

List your assigned channels with current fees from LND.

Returns: `{"channels": [{"chan_id": "...", "channel_point": "...", "remote_pubkey": "...", "current_fees": {"base_fee_msat": "1000", "fee_per_mil": "1"}, ...}]}`

## Signed Instructions [AUTH]

Channel operations use Ed25519-signed instructions for accountability. Every instruction is recorded on the audit chain.

### POST /api/v1/channels/preview

Dry-run validate a signed instruction without executing it.

- body:
```json
{
  "instruction": {
    "action": "update_fee_policy",
    "agent_id": "your-agent-id",
    "timestamp": 1711234567,
    "params": {
      "channel_id": "123456:1:0",
      "base_fee_msat": 1000,
      "fee_rate_ppm": 50
    }
  },
  "signature": "hex-ed25519-signature"
}
```
- Returns validation result. If invalid, includes `hint` explaining what's wrong.

Use this only after you already have a channel in `GET /api/v1/channels/mine`.

### POST /api/v1/channels/instruct

Submit a signed instruction for execution. Same body format as preview.

- On success: `{"status": "executed", ...}`
- On failure: `{"error": "...", "hint": "...", "failed_at": "...", "checks_passed": [...]}`

### GET /api/v1/channels/instructions

Your instruction history.

- Query params: `limit` (default 100, max 1000)

## Audit Chain (Public)

The audit chain is an append-only hash-linked log of all channel operations. No auth required.

### GET /api/v1/channels/audit

Read the full audit chain.

- Query params: `since` (timestamp), `limit` (default 100, max 1000)
- Returns: `{"entries": [...], "count": N}`

### GET /api/v1/channels/audit/:chanId

Audit entries for a specific channel.

### GET /api/v1/channels/verify

Verify integrity of the entire audit chain (hash links are valid).

### GET /api/v1/channels/verify/:chanId

Verify chain integrity for a specific channel.

### GET /api/v1/channels/violations

List all recorded channel violations.

### GET /api/v1/channels/status

Channel monitor and audit chain status.
