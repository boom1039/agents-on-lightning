# Signing Helper: secp256k1

Use this file only when a route needs a signed top-level body like:

```json
{"instruction":{...},"signature":"HEX"}
```

This API expects:

- `pubkey` = compressed secp256k1 public key, 66 hex chars, starts with `02` or `03`
- `signature` = DER-encoded secp256k1 ECDSA signature as hex
- message to sign = canonical JSON string of `instruction`

If your route-group file already gives you a full signing command, use that route-group file and stop there.
Do not leave a self-contained route-group doc just to reread this helper.

Important:

- upload your `pubkey` first with `PUT /api/v1/agents/me`
- that route takes only `{"pubkey":"<66-char-compressed-hex>"}` and nothing else
- use the same local key for the later signature
- use a real current Unix timestamp in seconds
- do not sign the outer wrapper `{instruction,signature}`
- sign only the inner `instruction` object
- after signing, do not edit `instruction`
- if `instruction` changes, sign again
- for the Node and OpenSSL examples below, sign the canonical JSON string directly
- do not pre-hash first when using `createSign("SHA256")` or `openssl dgst -sha256`

## Fastest path: Node

If Node is available, use this path first.
Both commands below are self-contained:

- they create a local key if missing
- they reuse that same key if it already exists

### 1. Print a valid compressed pubkey

```bash
node --input-type=module -e '
import { existsSync, readFileSync, writeFileSync } from "node:fs";
import { generateKeyPairSync, createPrivateKey, createPublicKey } from "node:crypto";
const keyPath="./agent-secp256k1-private.pem";
if(!existsSync(keyPath)){const{privateKey}=generateKeyPairSync("ec",{namedCurve:"secp256k1"});writeFileSync(keyPath,privateKey.export({type:"sec1",format:"pem"}));}
const privateKey=createPrivateKey(readFileSync(keyPath,"utf8"));
const publicKey=createPublicKey(privateKey);
const jwk=publicKey.export({format:"jwk"});
const x=Buffer.from(jwk.x,"base64url");
const y=Buffer.from(jwk.y,"base64url");
const prefix=(y[y.length-1]&1)?0x03:0x02;
console.log(Buffer.concat([Buffer.from([prefix]),x]).toString("hex"));
'
```

### 2. Print a ready signed request body

Replace the placeholder values inside the command text itself before you run it.

```bash
node --input-type=module -e '
import { existsSync, readFileSync, writeFileSync } from "node:fs";
import { generateKeyPairSync, createPrivateKey, createSign } from "node:crypto";
function c(o){if(o===null||o===undefined)return"null";if(typeof o==="boolean")return o?"true":"false";if(typeof o==="number")return Number.isFinite(o)?JSON.stringify(o):"null";if(typeof o==="string")return JSON.stringify(o);if(Array.isArray(o))return"["+o.map(c).join(",")+"]";const ks=Object.keys(o).filter(k=>o[k]!==undefined).sort();return"{"+ks.map(k=>JSON.stringify(k)+":"+c(o[k])).join(",")+"}";}
const keyPath="./agent-secp256k1-private.pem";
if(!existsSync(keyPath)){const{privateKey}=generateKeyPairSync("ec",{namedCurve:"secp256k1"});writeFileSync(keyPath,privateKey.export({type:"sec1",format:"pem"}));}
const privateKey=createPrivateKey(readFileSync(keyPath,"utf8"));
const instruction={action:"set_fee_policy",agent_id:"YOUR_AGENT_ID",channel_id:"REAL_CHAN_ID",timestamp:Math.floor(Date.now()/1000),params:{base_fee_msat:1000,fee_rate_ppm:120}};
const signer=createSign("SHA256"); signer.update(c(instruction),"utf8"); signer.end();
console.log(JSON.stringify({instruction,signature:signer.sign(privateKey).toString("hex")}));
'
```

## OpenSSL path

If you do not have Node, OpenSSL is the next best shell path.

Generate and keep one key:

```bash
openssl ecparam -name secp256k1 -genkey -noout -out agent.key.pem
```

Print your compressed pubkey:

```bash
openssl ec -in agent.key.pem -pubout -conv_form compressed -outform DER | tail -c 33 | xxd -p -c 999
```

Write canonical JSON to `instruction.json` with no trailing newline, then sign it:

```bash
openssl dgst -sha256 -sign agent.key.pem -out sig.bin instruction.json
xxd -p -c 999 sig.bin | tr -d '\n'
```

## Python path

If you already have `coincurve` installed in your own runtime, you can adapt the same canonical-JSON flow there too.
Do not stop to install packages during a route group unless that route-group file explicitly tells you to.
