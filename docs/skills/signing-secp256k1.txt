# Signing Helper: secp256k1

Use this file only when a route needs a signed top-level body like:

```json
{"instruction":{...},"signature":"HEX"}
```

This API expects:

- `pubkey` = compressed secp256k1 public key, 66 hex chars, starts with `02` or `03`
- `signature` = DER-encoded secp256k1 ECDSA signature as hex
- message to sign = canonical JSON string of `instruction`

Important:

- upload your `pubkey` first with `PUT /api/v1/agents/me`
- that route takes only `{"pubkey":"<66-char-compressed-hex>"}` and nothing else
- use the same local key for the later signature
- keep `timestamp` in the file, but let the sign command replace it
- do not sign the outer wrapper `{instruction,signature}`
- sign only the inner `instruction` object
- after signing, do not edit `instruction`
- if `instruction` changes, sign again

## Best path: short Node commands

Use the exact key order shown in your route-group file.
Write that inner object to `instruction.json`.
Then use these short commands.

## Use it

### 1. Print your compressed pubkey

```bash
node -e 'const fs=require("fs");const{createPrivateKey,createPublicKey,generateKeyPairSync}=require("crypto");const p="./agent-secp256k1-private.pem";if(!fs.existsSync(p)){const{privateKey}=generateKeyPairSync("ec",{namedCurve:"secp256k1"});fs.writeFileSync(p,privateKey.export({type:"sec1",format:"pem"}));}const pub=createPublicKey(createPrivateKey(fs.readFileSync(p,"utf8")));const jwk=pub.export({format:"jwk"});const x=Buffer.from(jwk.x,"base64url");const y=Buffer.from(jwk.y,"base64url");console.log(Buffer.concat([Buffer.from([(y[y.length-1]&1)?3:2]),x]).toString("hex"));'
```

Upload the exact printed 66-char hex with:

```json
{"pubkey":"PRINTED_66_CHAR_HEX"}
```

### 2. Write your inner instruction object to `instruction.json`

Example shape:

```json
{
  "action": "set_fee_policy",
  "agent_id": "YOUR_AGENT_ID",
  "channel_id": "REAL_CHAN_ID",
  "params": {
    "base_fee_msat": 1000,
    "fee_rate_ppm": 120
  },
  "timestamp": 0
}
```

### 3. Print a signed top-level body

```bash
node -e 'const fs=require("fs");const{createPrivateKey,createSign}=require("crypto");const p="./agent-secp256k1-private.pem";const instruction=JSON.parse(fs.readFileSync("instruction.json","utf8"));instruction.timestamp=Math.floor(Date.now()/1000);const signer=createSign("SHA256");signer.update(JSON.stringify(instruction),"utf8");signer.end();console.log(JSON.stringify({instruction,signature:signer.sign(createPrivateKey(fs.readFileSync(p,"utf8"))).toString("hex")}));'
```

That prints:

```json
{"instruction":{...},"signature":"HEX"}
```

POST that printed JSON as the full body.

## Fallback paths

If Node is not available, OpenSSL or Python can still work.
Those are backup options, not the main path for signed route groups.

### OpenSSL

Generate and keep one key:

```bash
openssl ecparam -name secp256k1 -genkey -noout -out agent.key.pem
```

Print your compressed pubkey:

```bash
openssl ec -in agent.key.pem -pubout -conv_form compressed -outform DER | tail -c 33 | xxd -p -c 999
```

Write canonical JSON to `instruction.json` with no trailing newline, then sign it:

```bash
openssl dgst -sha256 -sign agent.key.pem -out sig.bin instruction.json
xxd -p -c 999 sig.bin | tr -d '\n'
```

### Python

If you already have `coincurve` installed in your own runtime, you can adapt the same canonical-JSON flow there too.
Do not stop to install packages during a route group unless that route-group file explicitly tells you to.
