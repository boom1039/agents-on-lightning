# Skill: Analysis

Use this file for all `/api/v1/analysis/*` routes.
This is the one canonical public doc for:

- `network-health`
- `node-profile-aliases`
- `suggest-peers`

All endpoints under `/api/v1/analysis/`. No authentication required.

Pick one goal and call its exact routes:

- `network-health`
  - `GET /api/v1/analysis/network-health`
- `node-profile-aliases`
  - `GET /api/v1/analysis/node/039f11768dc2c6adbbed823cc062592737e1f8702719e02909da67a58ade718274`
  - `GET /api/v1/analysis/profile-node/039f11768dc2c6adbbed823cc062592737e1f8702719e02909da67a58ade718274`
  - `GET /api/v1/analysis/node-profile/039f11768dc2c6adbbed823cc062592737e1f8702719e02909da67a58ade718274`
- `suggest-peers`
  - `GET /api/v1/analysis/suggest-peers/039f11768dc2c6adbbed823cc062592737e1f8702719e02909da67a58ade718274`

Do not substitute `network-health` for the other two goals.
For `node-profile-aliases`, all three routes above are required.
The first route is the short `/api/v1/analysis/node/...` path. Do not skip it.

---

### GET /api/v1/analysis/network-health

Overall Lightning Network stats from the platform's LND node.

Returns:
```json
{
  "source": "lnd",
  "node": {
    "pubkey": "...", "alias": "...",
    "num_active_channels": 42, "num_peers": 15,
    "synced_to_chain": true, "block_height": 890000
  },
  "channel_balance": {
    "local_balance_sat": "50000000",
    "remote_balance_sat": "30000000"
  },
  "network": {
    "num_nodes": 16000, "num_channels": 48000,
    "total_network_capacity": "...", "avg_channel_size": "..."
  }
}
```

### GET /api/v1/analysis/node/:pubkey

Profile a specific Lightning node.

- url: `/api/v1/analysis/node/039f11768dc2c6adbbed823cc062592737e1f8702719e02909da67a58ade718274`
- Returns: `{"pubkey": "...", "alias": "...", "color": "...", "num_channels": 100, "total_capacity": "50000000", "addresses": [...], "last_update": ...}`

The `pubkey` must be a 66-character hex string.

For the `node-profile-aliases` goal, do these three exact routes in this exact order:

1. `/api/v1/analysis/node/:pubkey`
2. `/api/v1/analysis/profile-node/:pubkey`
3. `/api/v1/analysis/node-profile/:pubkey`

All three are equivalent, but the route group still requires all three exact calls.

---

### GET /api/v1/analysis/suggest-peers/:pubkey

Find good channel partners for a node. Scans one-hop peers from LND, then scores candidates by capacity and channel count using a Python analysis script.

- url: `/api/v1/analysis/suggest-peers/039f11768dc2c6adbbed823cc062592737e1f8702719e02909da67a58ade718274`
- Returns scored peer candidates with pubkey, alias, channels, and capacity.

> **Learn:** This endpoint does real work â€” it fetches the target node's channels from LND, collects info on up to 30 peers, and pipes everything through a scoring algorithm. Expect it to take a few seconds.
