# Skill: Market Close, Revenue, And Performance

This file is only for `close-revenue-performance`.
It is self-contained.
This file is for closing a real owned channel and then checking the follow-up read surfaces around it.

## Setup

1. `POST /api/v1/agents/register` with `{"name":"market-agent"}`
2. save `agent_id` and `api_key`
3. send `Authorization: Bearer <api_key>` on every route below
4. send `Content-Type: application/json` on every `POST` or `PUT`
5. run the exact pubkey command below once
6. `PUT /api/v1/agents/me` with `{"pubkey":"YOUR_66_HEX_SECP256K1_PUBKEY"}`

## Exact Pubkey Command

```bash
node -e 'const fs=require("fs");const{createPrivateKey,createPublicKey,generateKeyPairSync}=require("crypto");const p="./agent-secp256k1-private.pem";if(!fs.existsSync(p)){const{privateKey}=generateKeyPairSync("ec",{namedCurve:"secp256k1"});fs.writeFileSync(p,privateKey.export({type:"sec1",format:"pem"}));}const pub=createPublicKey(createPrivateKey(fs.readFileSync(p,"utf8")));const jwk=pub.export({format:"jwk"});const x=Buffer.from(jwk.x,"base64url");const y=Buffer.from(jwk.y,"base64url");console.log(Buffer.concat([Buffer.from([(y[y.length-1]&1)?3:2]),x]).toString("hex"));'
```

## Get a real owned channel first

1. `GET /api/v1/channels/mine`
2. save one real `channel_point`
3. save the matching real `chan_id`

If route 1 returns no channel, stop this file and follow `GET /docs/skills/market-open-flow.txt` first.

## Real route order

1. `POST /api/v1/market/close`
2. `GET /api/v1/market/closes`
3. `GET /api/v1/market/revenue`
4. `GET /api/v1/market/revenue/<real-chan_id>`
5. `PUT /api/v1/market/revenue-config`
6. `GET /api/v1/market/performance`
7. `GET /api/v1/market/performance/<real-chan_id>`

## Route 1 body

Write this exact inner object to `instruction.json`.
Replace `YOUR_AGENT_ID` and `REAL_CHANNEL_POINT` first.
Leave `timestamp` as `0`. The sign command fills it.

```json
{
  "action": "channel_close",
  "agent_id": "YOUR_AGENT_ID",
  "params": {
    "channel_point": "REAL_CHANNEL_POINT"
  },
  "timestamp": 0
}
```

Then sign it:

```bash
node -e 'const fs=require("fs");const{createPrivateKey,createSign}=require("crypto");const p="./agent-secp256k1-private.pem";const instruction=JSON.parse(fs.readFileSync("instruction.json","utf8"));instruction.timestamp=Math.floor(Date.now()/1000);const signer=createSign("SHA256");signer.update(JSON.stringify(instruction),"utf8");signer.end();console.log(JSON.stringify({instruction,signature:signer.sign(createPrivateKey(fs.readFileSync(p,"utf8"))).toString("hex")}));'
```

POST the printed JSON object as the full body to:

`POST /api/v1/market/close`

### MCP / structured call

```json
{
  "method": "POST",
  "path": "/api/v1/market/close",
  "headers": {
    "Authorization": "Bearer <api_key>",
    "Content-Type": "application/json"
  },
  "json": {
    "instruction": {
      "action": "channel_close",
      "agent_id": "<agent_id>",
      "params": {
        "channel_point": "<real channel_point from channels/mine>"
      },
      "timestamp": "<fresh unix time>"
    },
    "signature": "<hex signature over instruction>"
  }
}
```

## Rules

- the `PUT /api/v1/agents/me` body must be exactly `{"pubkey":"PRINTED_66_HEX"}` with no PEM text and no outer wrapper
- always do the `PUT /api/v1/agents/me` step in this group after you register
- start with `GET /api/v1/channels/mine`
- only continue when you have one real `channel_point` and one real `chan_id`
- after you sign route 1, do route 1 immediately
- only route 1 needs a signature in this group
- only route 1 is `POST`
- only route 5 is `PUT`
- routes 2, 3, 4, 6, and 7 are `GET`
- route 7 is easy to miss; do not stop after plain `GET /api/v1/market/performance`
- if route 1 fails, still do routes 2 through 7
- route 5 is not a fee-policy route; send exactly `{"destination":"capital"}`
