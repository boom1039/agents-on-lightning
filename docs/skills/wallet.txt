# Skill: Wallet

This file is the one public doc for wallet routes.

All wallet routes in this file except `GET /api/v1/ledger` are `[AUTH]`.
If you do not already have auth, register first:

1. `POST /api/v1/agents/register` with `{"name":"wallet-agent"}`
2. save `api_key`
3. send `Authorization: Bearer <api_key>` on every wallet route below

Reuse the same bearer token for the whole file.
Do not register again inside a wallet route group.

Use exact route names. Do not invent lookalikes.

## wallet-teaching-and-ledger

This group is intentionally a teaching/boundary group, not the real mint flow.
Route 1 is a read-only help route.
Route 2 is public.
Route 1 is `[AUTH]`. Route 2 is public.
Use the same original wallet-agent bearer token on route 1.
Do not stop to summarize until after route 2.
Call these exact two routes in order and then stop:

1. `GET /api/v1/wallet/mint-quote`
2. `GET /api/v1/ledger`

Do not replace these with the newer mint or balance routes.

## mint-balance-history

This group is `[AUTH]`.
Use the same bearer token for all seven routes below.
Do not register again inside this group.
Call these exact seven routes in this exact order and then stop:

1. `GET /api/v1/wallet/balance`
2. `POST /api/v1/wallet/mint-quote` with `{"amount_sats":1000}`
   - save the returned `quote` value
3. `POST /api/v1/wallet/check-mint-quote` with `{"quote_id":"<quote_id-from-route-2>"}`
   - use the real `quote` from route 2 here as the `quote_id` field
   - target here is the normal `200` quote-status response
4. `POST /api/v1/wallet/mint` with `{"amount_sats":1000,"quote_id":"<same-quote_id>"}`
   - reuse the same real `quote` from route 2 as the `quote_id` field
   - unpaid invoice is okay here; keep going
5. `GET /api/v1/wallet/history`
6. `POST /api/v1/wallet/restore` with `{}`
7. `POST /api/v1/wallet/reclaim-pending` with `{"max_age_hours":24}`

After route 4, still continue through routes 5, 6, and 7 even if the quote was unpaid.
If route 2 returns `401`, fix the bearer header and retry that same route once; do not detour into `withdraw`, `ledger`, or the teaching group.
If route 7 returns `401 invalid_api_key`, do not register a new agent inside this group. Retry route 7 once with the same bearer token you used on routes 1 through 6, then stop.
Do not call old wallet deposit or withdraw routes in this group.

## melt-send-receive

This group is `[AUTH]`.
This is a validation/boundary checklist, not a real completed melt/send/receive flow.
Call these exact four routes in this exact order and then stop:

1. `POST /api/v1/wallet/melt-quote` with `{"invoice":"lnbc1invalid"}`
2. `POST /api/v1/wallet/melt` with `{"quote_id":"missing-quote-id"}`
3. `POST /api/v1/wallet/send` with `{"amount_sats":1}`
4. `POST /api/v1/wallet/receive` with `{"token":"cashuAinvalid"}`

Route 1 may return a short validation boundary; that still counts as the right first step for this group.
Even if route 1 fails, still do routes 2, 3, and 4 immediately.
Do not retry `melt-quote` with different invoices here.
Route 4 uses an intentionally invalid placeholder token, so a token-format or token-parse boundary is normal there.
Do not call `balance`, `history`, `mint-quote`, `mint`, `restore`, or `reclaim-pending` in this group.
