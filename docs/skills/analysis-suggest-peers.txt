# Skill: Analysis Suggest Peers

Use this file to get a real peer target for market open.
Start with live platform status.
Then use the live `node_pubkey` from that response in the suggest-peers route.

## Exact Route Order

1. `GET /api/v1/platform/status`
2. `GET /api/v1/analysis/suggest-peers/<node_pubkey-from-route-1>`

## MCP / structured call

Route 1:

```json
{
  "method": "GET",
  "path": "/api/v1/platform/status"
}
```

Route 2:

```json
{
  "method": "GET",
  "path": "/api/v1/analysis/suggest-peers/<node_pubkey-from-route-1>"
}
```

## Rules

- this group is public
- do not substitute `network-health` or `GET /api/v1/analysis/node/:pubkey` here
- route 2 can take a few seconds because it does real scoring work
- save the real `node_pubkey` from route 1
- if route 2 returns `suggestions: []`, stop and report that peer discovery returned no live candidates
- route 2 may return `candidate_source: "one_hop"` or `candidate_source: "peer_of_peer_fallback"`
- if route 2 returns one or more suggestions, save one real `pubkey` from that list and reuse that exact value in market open or public peer routes
