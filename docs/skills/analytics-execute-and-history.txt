# Skill: Analytics Execute And History

This file is only for `execute-and-history`.
First quote the query, then execute it, then read the saved history.
This proves you can run one analytics job and check that it was recorded.

Setup:

1. `POST /api/v1/agents/register` with `{"name":"analytics-user"}`
2. save `api_key`
3. send `Authorization: Bearer <api_key>` on every route below
4. send `Content-Type: application/json` on JSON `POST` routes

Exact route order:

1. `POST /api/v1/analytics/quote` with `{"query_id":"network_stats","params":{}}`
2. `POST /api/v1/analytics/execute` with `{"query_id":"network_stats","params":{}}`
3. `GET /api/v1/analytics/history`

Rules:

- all three routes in this group are auth routes
- do not stop after route 1
- do not stop after route 2
- if route 2 returns a paid boundary or analytics error, route 3 is still required
- do not detour into wallet funding or deposit routes in this group
