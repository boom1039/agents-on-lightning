# Skill: Market Open Flow

This file is only for `open-flow`.
It is self-contained.

## Setup

1. `POST /api/v1/agents/register` with `{"name":"market-agent"}`
2. save `agent_id` and `api_key`
3. send `Authorization: Bearer <api_key>` on every route below
4. send `Content-Type: application/json` on every `POST` or `PUT`
5. generate one local secp256k1 signing key
6. `PUT /api/v1/agents/me` with `{"pubkey":"YOUR_66_HEX_SECP256K1_PUBKEY"}`

This file is self-contained.
Do not leave this file for another signing helper during this group.
Do not skip the pubkey upload.

## Exact Route Order

1. `POST /api/v1/market/preview`
2. `POST /api/v1/market/open`
3. `GET /api/v1/market/pending`

## Exact Signed Body Shape For Routes 1 And 2

```json
{
  "instruction": {
    "action": "channel_open",
    "agent_id": "YOUR_AGENT_ID",
    "timestamp": 1742400000,
    "params": {
      "peer_pubkey": "02aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa",
      "local_funding_amount_sats": 100000,
      "base_fee_msat": 1000,
      "fee_rate_ppm": 120,
      "min_htlc_msat": 1000,
      "time_lock_delta": 40,
      "max_htlc_msat": 50000000
    }
  },
  "signature": "YOUR_DER_SIGNATURE_HEX"
}
```

## Rules

- route 1 is `POST /api/v1/market/preview`
- route 2 is `POST /api/v1/market/open`
- route 3 is `GET /api/v1/market/pending`
- use the exact action string `"channel_open"`
- use the exact funding key `"local_funding_amount_sats"`
- keep `local_funding_amount_sats` at `100000` for this quick group
- sign the inner `instruction` object only
- build the route 1 body fresh and sign it once
- build the route 2 body fresh, refresh `timestamp`, and sign again
- do not reuse the exact same signed body for both routes
- do not leave the signing example action as `set_fee_policy`
- if route 1 says your pubkey is missing or the signature is invalid, rebuild the signed body once and resend route 1
- if route 1 says `Insufficient available balance`, still do routes 2 and 3
- if route 2 says `Insufficient available balance`, still do route 3
- do not detour into capital deposit, deposit-address, wallet funding, or extra market reads in this group

## Exact Pubkey Command

```bash
node --input-type=module -e '
import { existsSync, readFileSync, writeFileSync } from "node:fs";
import { generateKeyPairSync, createPrivateKey, createPublicKey } from "node:crypto";
const keyPath="./agent-secp256k1-private.pem";
if(!existsSync(keyPath)){const{privateKey}=generateKeyPairSync("ec",{namedCurve:"secp256k1"});writeFileSync(keyPath,privateKey.export({type:"sec1",format:"pem"}));}
const privateKey=createPrivateKey(readFileSync(keyPath,"utf8"));
const publicKey=createPublicKey(privateKey);
const jwk=publicKey.export({format:"jwk"});
const x=Buffer.from(jwk.x,"base64url");
const y=Buffer.from(jwk.y,"base64url");
const prefix=(y[y.length-1]&1)?0x03:0x02;
console.log(Buffer.concat([Buffer.from([prefix]),x]).toString("hex"));
'
```

Pubkey upload rule:

- that command prints one compressed secp256k1 pubkey only
- it must be 66 hex chars and start with `02` or `03`
- send only this body to `PUT /api/v1/agents/me`:

```json
{"pubkey":"PASTE_THE_ONE_LINE_66_CHAR_HEX_HERE"}
```

- do not send PEM text
- do not send the signed request body to `PUT /api/v1/agents/me`

## Exact Signed `channel_open` Body Command

```bash
node --input-type=module -e '
import { existsSync, readFileSync, writeFileSync } from "node:fs";
import { generateKeyPairSync, createPrivateKey, createSign } from "node:crypto";
function c(o){if(o===null||o===undefined)return"null";if(typeof o==="boolean")return o?"true":"false";if(typeof o==="number")return Number.isFinite(o)?JSON.stringify(o):"null";if(typeof o==="string")return JSON.stringify(o);if(Array.isArray(o))return"["+o.map(c).join(",")+"]";const ks=Object.keys(o).filter(k=>o[k]!==undefined).sort();return"{"+ks.map(k=>JSON.stringify(k)+":"+c(o[k])).join(",")+"}";}
const keyPath="./agent-secp256k1-private.pem";
if(!existsSync(keyPath)){const{privateKey}=generateKeyPairSync("ec",{namedCurve:"secp256k1"});writeFileSync(keyPath,privateKey.export({type:"sec1",format:"pem"}));}
const privateKey=createPrivateKey(readFileSync(keyPath,"utf8"));
const instruction={action:"channel_open",agent_id:"YOUR_AGENT_ID",timestamp:Math.floor(Date.now()/1000),params:{peer_pubkey:"02aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa",local_funding_amount_sats:100000,base_fee_msat:1000,fee_rate_ppm:120,min_htlc_msat:1000,time_lock_delta:40,max_htlc_msat:50000000}};
const signer=createSign("SHA256"); signer.update(c(instruction),"utf8"); signer.end();
console.log(JSON.stringify({instruction,signature:signer.sign(privateKey).toString("hex")}));
'
```

Before you run that command:

- replace `YOUR_AGENT_ID` with the real returned `agent_id`
- keep the action exactly `channel_open`
- for route 2, rerun it fresh so `timestamp` changes
