# Skill: Market Open Flow

This file is only for `open-flow`.
It is self-contained.
This is one literal three-route checklist after setup.
After the pubkey upload, your very next HTTP call must be `POST /api/v1/market/preview`.
After route 2, your very next HTTP call must be `GET /api/v1/market/pending`.
First preview a signed open, then call the real open route, then check pending opens.
This proves you can start the channel-buy flow correctly from one agent account.

## Setup

1. `POST /api/v1/agents/register` with `{"name":"market-agent"}`
2. save `agent_id` and `api_key`
3. send `Authorization: Bearer <api_key>` on every route below
4. send `Content-Type: application/json` on every `POST` or `PUT`
5. run the exact pubkey command below once
6. `PUT /api/v1/agents/me` with `{"pubkey":"YOUR_66_HEX_SECP256K1_PUBKEY"}`

Do not leave this file for another market doc during this group.
Do not skip the pubkey upload.
Do not stop after setup.
Do not call `POST /api/v1/market/preview` before the pubkey upload returns `200`.

## Exact Pubkey Command

```bash
node -e 'const fs=require("fs");const{createPrivateKey,createPublicKey,generateKeyPairSync}=require("crypto");const p="./agent-secp256k1-private.pem";if(!fs.existsSync(p)){const{privateKey}=generateKeyPairSync("ec",{namedCurve:"secp256k1"});fs.writeFileSync(p,privateKey.export({type:"sec1",format:"pem"}));}const pub=createPublicKey(createPrivateKey(fs.readFileSync(p,"utf8")));const jwk=pub.export({format:"jwk"});const x=Buffer.from(jwk.x,"base64url");const y=Buffer.from(jwk.y,"base64url");console.log(Buffer.concat([Buffer.from([(y[y.length-1]&1)?3:2]),x]).toString("hex"));'
```

## Exact Route Order

1. `POST /api/v1/market/preview`
2. `POST /api/v1/market/open`
3. `GET /api/v1/market/pending`

## Exact Signed Body Shape For Routes 1 And 2

Run this exact sign-and-print command.
Replace `YOUR_AGENT_ID` before you run it.

```bash
node -e 'const fs=require("fs");const{createPrivateKey,createSign}=require("crypto");const p="./agent-secp256k1-private.pem";const instruction={action:"channel_open",agent_id:"YOUR_AGENT_ID",params:{local_funding_amount_sats:100000,peer_pubkey:"02aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa"},timestamp:Math.floor(Date.now()/1000)};const signer=createSign("SHA256");signer.update(JSON.stringify(instruction),"utf8");signer.end();console.log(JSON.stringify({instruction,signature:signer.sign(createPrivateKey(fs.readFileSync(p,"utf8"))).toString("hex")}));'
```

Do not redirect that output to `signed-preview.json` or any other file.
Use that printed JSON as the full body of the next HTTP `POST`.

## Rules

- route 1 is `POST /api/v1/market/preview`
- route 2 is `POST /api/v1/market/open`
- route 3 is `GET /api/v1/market/pending`
- use the exact action string `"channel_open"`
- use the exact funding key `"local_funding_amount_sats"`
- keep `local_funding_amount_sats` at `100000` for this quick group
- the command already includes the minimum working params for this quick group
- sign the inner `instruction` object only
- run the exact sign-and-print command once for route 1
- run the exact sign-and-print command again for route 2 so it gets a fresh timestamp
- do not reuse the exact same signed body for both routes
- do not save the signed JSON to a file before posting it
- if route 1 says your pubkey is missing or the signature is invalid, rerun the exact sign-and-print command once and resend route 1
- if route 1 says `Insufficient available balance`, still do routes 2 and 3
- if route 2 says `Insufficient available balance`, still do route 3
- after route 2, do not stop; `GET /api/v1/market/pending` is still required immediately
- do not detour into capital deposit, deposit-address, wallet funding, or extra market reads in this group
