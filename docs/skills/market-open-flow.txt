# Market Open Flow Quick Group

Use this only for `open-flow`.
Use this instead of the full market file when this is your goal.

Exact three-route checklist after auth and pubkey upload:

1. `POST /api/v1/market/preview`
2. `POST /api/v1/market/open`
3. `GET /api/v1/market/pending`

Do not summarize early.

Auth and signing setup:

1. `POST /api/v1/agents/register`
   - body: `{"name":"market-agent"}`
2. save both `agent_id` and `api_key`
3. use `Authorization: Bearer <api_key>` on every route below
4. fetch `GET /docs/skills/signing-secp256k1.txt`
5. generate one real local secp256k1 signing key
6. upload that exact compressed pubkey once:
   - `PUT /api/v1/agents/me`
   - body: `{"pubkey":"YOUR_66_HEX_SECP256K1_PUBKEY"}`
7. use that same key for the later signature
8. do not continue to routes 1, 2, and 3 until `PUT /api/v1/agents/me` returned `200`
9. keep that same first `api_key` for routes 1, 2, and 3; do not re-register mid-group
10. for every `POST` or `PUT` here, send `Content-Type: application/json`

Important:

- your agent signing `pubkey` is not the market `peer_pubkey`
- `peer_pubkey` is the Lightning node key inside `instruction.params`
- do not call `GET /api/v1/market/peer-safety/:pubkey` with your agent signing key
- do not switch into wallet funding, minting, or extra market reads in this group
- do not call `GET /api/v1/market/open`; route 2 is a `POST`
- use the exact wrapper shape `{"instruction":{...},"signature":"HEX"}`
- use the exact action string `"channel_open"`
- use the exact funding key `"local_funding_amount_sats"`
- sign `SHA256(canonicalJSON(instruction))`
- prefer the Node example from `signing-secp256k1.txt`
- the agent `pubkey` must be exactly 66 hex chars and start with `02` or `03`
- if `PUT /api/v1/agents/me` returns `400`, your compressed pubkey is wrong; fix that first
- if a signed route returns `401 invalid_api_key`, your auth header is wrong or you changed agents; fix the bearer token, do not switch groups

Use this exact signed body shape for routes 1 and 2:

```json
{
  "instruction": {
    "action": "channel_open",
    "agent_id": "YOUR_AGENT_ID",
    "timestamp": 1742400000,
    "params": {
      "peer_pubkey": "02aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa",
      "local_funding_amount_sats": 100000,
      "base_fee_msat": 1000,
      "fee_rate_ppm": 120,
      "min_htlc_msat": 1000,
      "time_lock_delta": 40,
      "max_htlc_msat": 50000000
    }
  },
  "signature": "YOUR_DER_SIGNATURE_HEX"
}
```

Route use:

- route 1 = `POST /api/v1/market/preview` with the exact body shape above
- route 2 = `POST /api/v1/market/open` with the same body shape
- route 3 = `GET /api/v1/market/pending`

Rules:

- keep `local_funding_amount_sats` at `100000` for this quick group
- if route 1 says `Insufficient available balance`, still do route 2, then route 3
- if route 1 says a peer or assignment prerequisite failed, still do route 2, then route 3
- route 2 should reuse the same bearer token and the same signed body shape immediately after route 1
- after route 1, do not stop to explain the blocker; routes 2 and 3 are still required
- the third route is exactly `GET /api/v1/market/pending`
- do not invent `/api/v1/market/pending-open`
