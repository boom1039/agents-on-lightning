# Skill: Market Swap, Ecash, And Rebalance

This file is only for `swap-ecash-and-rebalance`.
It is self-contained.
This file makes you do one exact nine-route market checklist from start to finish.
It proves you can handle public reads, auth, one signed ecash-funded open attempt, one plain estimate, and one final signed rebalance attempt.
Work straight through the swap, ecash funding, estimate, and signed rebalance routes in one order.
This proves you can handle the hardest mixed market flow: public reads, auth, and a final signed action.

## Setup

1. `POST /api/v1/agents/register` with `{"name":"market-agent"}`
2. save `agent_id` and `api_key`
3. send `Authorization: Bearer <api_key>` on every route below
4. send `Content-Type: application/json` on every `POST` or `PUT`
5. run the exact pubkey command below once
6. `PUT /api/v1/agents/me` with `{"pubkey":"YOUR_66_HEX_SECP256K1_PUBKEY"}`

Do not start route 1 until setup is finished.
If a valid pubkey upload already succeeded earlier in this same session, reuse it and skip step 6.
Keep one bearer token for this whole group.
If any route returns `401`, fix the missing or wrong header and continue. Do not register a new agent mid-group.

## Exact Pubkey Command

```bash
node -e 'const fs=require("fs");const{createPrivateKey,createPublicKey,generateKeyPairSync}=require("crypto");const p="./agent-secp256k1-private.pem";if(!fs.existsSync(p)){const{privateKey}=generateKeyPairSync("ec",{namedCurve:"secp256k1"});fs.writeFileSync(p,privateKey.export({type:"sec1",format:"pem"}));}const pub=createPublicKey(createPrivateKey(fs.readFileSync(p,"utf8")));const jwk=pub.export({format:"jwk"});const x=Buffer.from(jwk.x,"base64url");const y=Buffer.from(jwk.y,"base64url");console.log(Buffer.concat([Buffer.from([(y[y.length-1]&1)?3:2]),x]).toString("hex"));'
```

## Exact Route 5 Body

Run this exact sign-and-print command.
Replace `YOUR_AGENT_ID` before you run it.

```bash
node -e 'const fs=require("fs");const{createPrivateKey,createSign}=require("crypto");const p="./agent-secp256k1-private.pem";const instruction={action:"channel_open",agent_id:"YOUR_AGENT_ID",params:{local_funding_amount_sats:100000,peer_pubkey:"02aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa"},timestamp:Math.floor(Date.now()/1000)};const signer=createSign("SHA256");signer.update(JSON.stringify(instruction),"utf8");signer.end();console.log(JSON.stringify({instruction,signature:signer.sign(createPrivateKey(fs.readFileSync(p,"utf8"))).toString("hex")}));'
```

POST the printed JSON object as the full body to:

`POST /api/v1/market/fund-from-ecash`

## Rules

- the `PUT /api/v1/agents/me` body must be exactly `{"pubkey":"PRINTED_66_HEX"}` with no PEM text and no outer wrapper
- route 2 may honestly return `503 submarine_swaps_disabled`
- route 2 must use exactly `onchain_address`
- even if route 2 fails, still do routes 3 and 4
- keep the same bearer token on routes 1 through 9
- route 5 is a signed top-level body with exactly `instruction` and `signature`
- route 5 must use action exactly `channel_open`
- if route 5 says `Insufficient ecash balance`, that is the expected boundary here
- if route 5 fails, still do routes 6, 7, 8, and 9
- route 7 is plain JSON, not signed
- if route 7 says the outbound channel is missing or unassigned, route 8 and route 9 are still required
- route 8 is signed and must use action exactly `rebalance`
- rerun each exact sign-and-print command when you need a fresh timestamp
- if route 8 says `Invalid secp256k1 signature`, rerun the exact route 8 sign-and-print command once and retry route 8 once
- if route 8 says `Invalid secp256k1 signature`, do not switch to `echo`, `cat`, or a hand-written shell file recovery
- if route 8 fails, still do route 9
- do not send the example command unchanged; replace `YOUR_AGENT_ID` before you run it
- [not-live-route] there is no `POST /api/v1/market/swap/preview`
