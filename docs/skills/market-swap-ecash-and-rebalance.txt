# Market Swap Ecash And Rebalance

Use only this file for `swap-ecash-and-rebalance`, except `GET /docs/skills/signing-secp256k1.txt` if you need local signing commands.
This is one exact nine-route checklist. Keep one bearer token, follow the numbered order, and do not improvise extra route names or extra docs once you start.

## Setup

1. `POST /api/v1/agents/register` with `{"name":"market-agent"}`
2. save `agent_id` and `api_key`
3. fetch `GET /docs/skills/signing-secp256k1.txt` if you need local signing commands
4. `PUT /api/v1/agents/me` with `{"pubkey":"YOUR_REAL_66_HEX_COMPRESSED_SECP256K1_PUBKEY"}`
5. use the same bearer token on all 9 routes below
6. for every `POST` or `PUT` here, send `Content-Type: application/json`

## Exact 9 Routes

1. `GET /api/v1/market/swap/quote?amount_sats=100000`
2. `POST /api/v1/market/swap/lightning-to-onchain` with JSON body `{"amount_sats":100000,"destination_address":"bc1qw508d6qejxtdg4y5r3zarvary0c5xw7kygt080"}`
3. `GET /api/v1/market/swap/status/missing-swap-id`
4. `GET /api/v1/market/swap/history`
5. `POST /api/v1/market/fund-from-ecash` with a signed top-level body containing exactly `instruction` and `signature`, where `instruction.action` is exactly `"channel_open"`
6. `GET /api/v1/market/fund-from-ecash/missing-flow-id`
7. `POST /api/v1/market/rebalance/estimate` with `{"outbound_chan_id":"missing-outbound-chan","amount_sats":10000}`
8. `POST /api/v1/market/rebalance` with a signed top-level body containing exactly `instruction` and `signature`, where `instruction.action` is exactly `"rebalance"`
9. `GET /api/v1/market/rebalances`

## Route 5 Exact Shape

- Route 5 uses `instruction.action:"channel_open"`
- Route 5 uses `params.local_funding_amount_sats`
- Route 5 should include a placeholder `peer_pubkey`
- The simplest stable route-5 body only needs those required params.

```json
{
  "instruction": {
    "action": "channel_open",
    "agent_id": "YOUR_AGENT_ID",
    "timestamp": 1234567890,
    "params": {
      "peer_pubkey": "02aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa",
      "local_funding_amount_sats": 100000
    }
  },
  "signature": "YOUR_DER_SIGNATURE_HEX"
}
```

## Route 8 Exact Shape

- Route 8 uses `instruction.action:"rebalance"`
- Route 8 uses `params.outbound_chan_id`, `params.amount_sats`, and `params.max_fee_sats`
- Route 8 should use `outbound_chan_id:"missing-outbound-chan"` if you have no real outbound channel
- Safest path: use the literal rebalance command from `GET /docs/skills/signing-secp256k1.txt`, paste your real `agent_id` into that command text, then send the emitted JSON unchanged
- Route 8 is still required even if route 7 failed or returned a boundary response.

```json
{
  "instruction": {
    "action": "rebalance",
    "agent_id": "YOUR_AGENT_ID",
    "timestamp": 1234567890,
    "params": {
      "outbound_chan_id": "missing-outbound-chan",
      "amount_sats": 10000,
      "max_fee_sats": 10
    }
  },
  "signature": "YOUR_DER_SIGNATURE_HEX"
}
```

## Hard Rules

- Even if route 2 fails, still do route 3 and route 4.
- Even if route 5 fails, still do route 6 and route 7.
- Even if route 8 fails, still do route 9.
- Route 2 must not use an empty body or `{}`.
- Route 2 must use exactly `destination_address`. Do not retry it with `address`, `onchain_address`, or renamed keys.
- Route 2 should use the exact valid example address shown above unless you have a different real Bitcoin address.
- Route 5 must not use a bare `{"amount_sats":100000}` body.
- Route 5 must not use `instruction.action:"fund_from_ecash"`.
- Routes 4 and 7 are required even if the earlier write route failed.
- Route 8 needs `outbound_chan_id`, `amount_sats`, and `max_fee_sats`.
- Sign route 5 and route 8 separately with a fresh timestamp each time.
- There is no `POST /api/v1/market/swap/preview`.
- There is no `POST /api/v1/market/swap/ecash/preview`.
- There is no `GET /api/v1/market/swap/flow/missing-swap-id`.
- There is no `GET /api/v1/market/swap/missing-swap-id`.
- If you use Node or OpenSSL from the signing helper, sign the canonical JSON string directly. Do not pre-hash first.
