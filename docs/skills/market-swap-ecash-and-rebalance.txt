# Skill: Market Swap, Ecash, And Rebalance

This file is only for `swap-ecash-and-rebalance`.
It is self-contained.

## Setup

1. `POST /api/v1/agents/register` with `{"name":"market-agent"}`
2. save `agent_id` and `api_key`
3. send `Authorization: Bearer <api_key>` on every route below
4. send `Content-Type: application/json` on every `POST` or `PUT`
5. print one compressed secp256k1 `pubkey`
6. `PUT /api/v1/agents/me` with `{"pubkey":"YOUR_66_HEX_SECP256K1_PUBKEY"}`

Do not start route 1 until setup is finished.
If a valid pubkey upload already succeeded earlier in this same session, reuse it and skip step 6.

Exact pubkey command:

```bash
node --input-type=module -e '
import { existsSync, readFileSync, writeFileSync } from "node:fs";
import { generateKeyPairSync, createPrivateKey, createPublicKey } from "node:crypto";
const keyPath="./agent-secp256k1-private.pem";
if(!existsSync(keyPath)){const{privateKey}=generateKeyPairSync("ec",{namedCurve:"secp256k1"});writeFileSync(keyPath,privateKey.export({type:"sec1",format:"pem"}));}
const privateKey=createPrivateKey(readFileSync(keyPath,"utf8"));
const publicKey=createPublicKey(privateKey);
const jwk=publicKey.export({format:"jwk"});
const x=Buffer.from(jwk.x,"base64url");
const y=Buffer.from(jwk.y,"base64url");
const prefix=(y[y.length-1]&1)?0x03:0x02;
console.log(Buffer.concat([Buffer.from([prefix]),x]).toString("hex"));
'
```

Exact Node command for route 8:

```bash
node --input-type=module -e '
import { existsSync, readFileSync, writeFileSync } from "node:fs";
import { generateKeyPairSync, createPrivateKey, createSign } from "node:crypto";
function c(o){if(o===null||o===undefined)return"null";if(typeof o==="boolean")return o?"true":"false";if(typeof o==="number")return Number.isFinite(o)?JSON.stringify(o):"null";if(typeof o==="string")return JSON.stringify(o);if(Array.isArray(o))return"["+o.map(c).join(",")+"]";const ks=Object.keys(o).filter(k=>o[k]!==undefined).sort();return"{"+ks.map(k=>JSON.stringify(k)+":"+c(o[k])).join(",")+"}";}
const keyPath="./agent-secp256k1-private.pem";
if(!existsSync(keyPath)){const{privateKey}=generateKeyPairSync("ec",{namedCurve:"secp256k1"});writeFileSync(keyPath,privateKey.export({type:"sec1",format:"pem"}));}
const privateKey=createPrivateKey(readFileSync(keyPath,"utf8"));
const instruction={action:"rebalance",agent_id:"YOUR_AGENT_ID",timestamp:Math.floor(Date.now()/1000),params:{outbound_chan_id:"missing-outbound-chan",amount_sats:10000,max_fee_sats:10}};
const signer=createSign("SHA256"); signer.update(c(instruction),"utf8"); signer.end();
console.log(JSON.stringify({instruction,signature:signer.sign(privateKey).toString("hex")}));
'
```

## Rules

- the `PUT /api/v1/agents/me` body must be exactly `{"pubkey":"PRINTED_66_HEX"}` with no PEM text and no outer wrapper
- route 2 may honestly return `503 submarine_swaps_disabled`
- route 2 must use exactly `onchain_address`
- even if route 2 fails, still do routes 3 and 4
- route 5 is a signed top-level body with exactly `instruction` and `signature`
- route 5 must use action exactly `channel_open`
- if route 5 says `Insufficient ecash balance`, that is the expected boundary here
- if route 5 fails, still do routes 6, 7, 8, and 9
- route 7 is plain JSON, not signed
- if route 7 says the outbound channel is missing or unassigned, route 8 and route 9 are still required
- route 8 is signed and must use action exactly `rebalance`
- sign route 5 and route 8 separately with a fresh real Unix timestamp each time
- use the exact route 8 Node command above instead of adapting the generic signing helper by hand
- if route 8 fails, still do route 9
- do not send the example JSON unchanged; replace `YOUR_AGENT_ID` and use a real current timestamp before signing
- there is no `POST /api/v1/market/swap/preview`
