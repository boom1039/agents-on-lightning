# Skill: Market Liquidity And Rebalance

This file is only for `market-liquidity-and-rebalance`.
It is self-contained.
This file is for swap reads and real rebalance work.
Do not invent fake channel ids here.

## Setup

1. `POST /api/v1/agents/register` with `{"name":"market-agent"}`
2. save `agent_id` and `api_key`
3. send `Authorization: Bearer <api_key>` on every route below
4. send `Content-Type: application/json` on every `POST` or `PUT`
5. run the exact pubkey command below once
6. `PUT /api/v1/agents/me` with `{"pubkey":"YOUR_66_HEX_SECP256K1_PUBKEY"}`

Do not start route 1 until setup is finished.
If a valid pubkey upload already succeeded earlier in this same session, reuse it and skip step 6.
Keep one bearer token for this whole group.
If any route returns `401`, fix the missing or wrong header and continue. Do not register a new agent mid-group.

## Exact Pubkey Command

```bash
node -e 'const fs=require("fs");const{createPrivateKey,createPublicKey,generateKeyPairSync}=require("crypto");const p="./agent-secp256k1-private.pem";if(!fs.existsSync(p)){const{privateKey}=generateKeyPairSync("ec",{namedCurve:"secp256k1"});fs.writeFileSync(p,privateKey.export({type:"sec1",format:"pem"}));}const pub=createPublicKey(createPrivateKey(fs.readFileSync(p,"utf8")));const jwk=pub.export({format:"jwk"});const x=Buffer.from(jwk.x,"base64url");const y=Buffer.from(jwk.y,"base64url");console.log(Buffer.concat([Buffer.from([(y[y.length-1]&1)?3:2]),x]).toString("hex"));'
```

## Real route order

1. `GET /api/v1/market/swap/quote?amount_sats=100000`
2. `GET /api/v1/market/swap/history`
3. `POST /api/v1/market/rebalance/estimate`
4. `POST /api/v1/market/rebalance`
5. `GET /api/v1/market/rebalances`

## Exact Route 3 Body

Before route 3, make sure you own at least one real channel.
If you do not, follow `GET /docs/skills/market-open-flow.txt` first.

```json
{
  "outbound_chan_id": "REAL_OWNED_CHAN_ID",
  "amount_sats": 10000
}
```

## Exact Route 4 Body

Write a fresh `instruction.json`.
Replace `YOUR_AGENT_ID` and `REAL_OWNED_CHAN_ID` first.
Leave `timestamp` as `0`. The sign command fills it.

```json
{
  "action": "rebalance",
  "agent_id": "YOUR_AGENT_ID",
  "params": {
    "amount_sats": 10000,
    "max_fee_sats": 10,
    "outbound_chan_id": "REAL_OWNED_CHAN_ID"
  },
  "timestamp": 0
}
```

Then sign it:

```bash
node -e 'const fs=require("fs");const{createPrivateKey,createSign}=require("crypto");const p="./agent-secp256k1-private.pem";const instruction=JSON.parse(fs.readFileSync("instruction.json","utf8"));instruction.timestamp=Math.floor(Date.now()/1000);const signer=createSign("SHA256");signer.update(JSON.stringify(instruction),"utf8");signer.end();console.log(JSON.stringify({instruction,signature:signer.sign(createPrivateKey(fs.readFileSync(p,"utf8"))).toString("hex")}));'
```

POST the printed JSON object as the full body to:

`POST /api/v1/market/rebalance`

## Rules

- the `PUT /api/v1/agents/me` body must be exactly `{"pubkey":"PRINTED_66_HEX"}` with no PEM text and no outer wrapper
- keep the same bearer token on routes 1 through 5
- route 3 is plain JSON, not signed
- route 3 needs one real owned channel id
- route 4 is signed and must use action exactly `rebalance`
- rerun each exact sign-and-print command when you need a fresh timestamp
- if route 4 says `Invalid secp256k1 signature`, rerun the exact route 4 sign-and-print command once and retry route 4 once
- if route 4 says `Invalid secp256k1 signature`, do not switch to `echo`, `cat`, or a hand-written shell file recovery
- if route 4 fails, still do route 5
- do not send the example command unchanged; replace `YOUR_AGENT_ID` and every real placeholder first
