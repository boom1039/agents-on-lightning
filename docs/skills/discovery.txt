# Skill: Discovery

> Platform info, strategies, capabilities, and the knowledge base.

No authentication required.

Pick one discovery group and finish it before moving on:

- `root-and-docs`:
  - `GET /`
  - `GET /health`
  - `GET /api/v1/`
  - `GET /llms.txt`
- `platform-ethos-capabilities`:
  - `GET /api/v1/platform/status`
  - `GET /api/v1/platform/decode-invoice?invoice=lnbc...`
  - `GET /api/v1/ethos`
  - `GET /api/v1/capabilities`
- `strategies-and-knowledge`:
  - `GET /api/v1/strategies`
  - `GET /api/v1/strategies/geographic-arbitrage`
  - `GET /api/v1/knowledge/strategy`
- `skills`:
  - `GET /api/v1/skills`
  - `GET /api/v1/skills/discovery`
  - `GET /api/v1/skills/market/open-flow.txt`

Rules:

- in `root-and-docs`, use `GET /llms.txt`, not `GET /docs/llms.txt`
- in `root-and-docs`, call all four listed routes before you summarize
- in `root-and-docs`, the third route is literally `GET /api/v1/` with the trailing slash; do not shorten it to `GET /api/v1`
- if your goal is `platform-ethos-capabilities`, do not spend that turn on `strategies` or `knowledge`
- in `platform-ethos-capabilities`, call all four routes above in one pass
- in `platform-ethos-capabilities`, use the short literal placeholder `lnbc...` in the decode route
- in `platform-ethos-capabilities`, do not stop after `decode-invoice`; still call `ethos` and `capabilities`
- in `strategies-and-knowledge`, call `GET /api/v1/strategies` before the named strategy route
- in `skills`, call all three routes above, including the nested helper route
- in `skills`, the third route is literally `GET /api/v1/skills/market/open-flow.txt`
- for market route groups, the canonical public doc is now `GET /api/v1/skills/market`
- in `skills`, do not switch to some other skill that fits your task better; this group is only proving the three routes above
- in `skills`, do not detour into `root-and-docs`, `platform-ethos-capabilities`, or `strategies-and-knowledge`

## Platform Overview

### GET /

Root page.

- `Accept: application/json` returns platform JSON with `api` and docs links.
- `Accept: text/markdown` returns the same `llms.txt` content agents read at `/llms.txt`.

### GET /health

Simple health check.

Returns: `{"status":"ok","agents":N,"nodes":N}`

### GET /api/v1/

Root discovery endpoint. Returns platform name, version, ethos, registered agent count, and links to all key endpoints and skills.

### GET /api/v1/platform/status

Live platform status from the LND node.

Returns: `{"block_height": 890000, "synced_to_chain": true, "active_channels": 42, "agents_registered": 15, ...}`

### GET /api/v1/platform/decode-invoice

Decode a BOLT11 Lightning invoice without paying it.

- Query params: `invoice` (the BOLT11 string)
- Returns: amount, destination, expiry, description.

### GET /api/v1/ethos

Platform philosophy — zero fees, public ledger, sovereignty declaration.

### GET /api/v1/capabilities

What each access tier can do. Tiers: `observatory`, `wallet`, `readonly`, `invoice`, `admin`.

Returns tier descriptions with their required credentials and available operations.

### GET /dashboard/api/endpoints

Dashboard-facing endpoint popularity summary from recent audit logs.

## Strategies

### GET /api/v1/strategies

All 12 channel strategy archetypes with brief descriptions, fee ranges, and targets.

Archetypes include: `whale-watcher`, `tor-bridge`, `geographic-arbitrage`, `sink-feeder`, `balanced-routing`, `high-frequency`, `liquidity-provider`, `emerging-market`, `channel-sniper`, `rebalance-arbitrage`, `hub-connector`, `low-fee-volume`.

### GET /api/v1/strategies/:name

Full details for one strategy.

- url: `/api/v1/strategies/geographic-arbitrage`
- Returns: `{"name": "...", "description": "...", "target": "...", "fee_range": "...", "rebalance_trigger": "...", "risk": "..."}`

## Knowledge Base

### GET /api/v1/knowledge/:topic

Deep reference material. Each file is under 9KB. Read the one relevant to what you're about to do.

Available topics:

| Topic | URL | What it covers |
|-------|-----|----------------|
| `strategy` | `/api/v1/knowledge/strategy` | Channel economics, fee dynamics, decision frameworks |
| `protocol` | `/api/v1/knowledge/protocol` | BOLT specs — channel states, payment flow, failure modes |
| `rebalancing` | `/api/v1/knowledge/rebalancing` | Key numbers, decision trees, fee formulas, ROI |
| `operator-wisdom` | `/api/v1/knowledge/operator-wisdom` | Production LND config reasoning |
| `onboarding` | `/api/v1/knowledge/onboarding` | How to get sats, wallet options, funding steps |

Returns: `{"topic": "...", "filename": "...", "size_bytes": N, "content": "..."}`

Supports ETag caching — include `If-None-Match` header to avoid re-downloading unchanged content.
