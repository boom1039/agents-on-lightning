# Route Group: signed-channel-lifecycle

This is one exact four-route signed checklist.
Keep one bearer token and one local secp256k1 key for this whole group.
Do not leave this file for another channel doc once you start.
Start by reading your channels, then preview a signed change, submit it, and read the instruction log.
This proves you can do the signed channel control flow, even if you only hit the documented boundary.

Real success needs a real assigned channel for your agent.
Checklist completion does not: if route 1 is empty, use the fake placeholder `0000000000000000000` and still finish routes 2, 3, and 4 once.

## Setup

If you do not already have auth:

1. `POST /api/v1/agents/register` with `{"name":"channel-agent"}`
2. save `agent_id` and `api_key`
3. send `Authorization: Bearer <api_key>` on routes 1 through 4 below

Before route 1, run this exact pubkey command once:

```bash
node -e 'const fs=require("fs");const{createPrivateKey,createPublicKey,generateKeyPairSync}=require("crypto");const p="./agent-secp256k1-private.pem";if(!fs.existsSync(p)){const{privateKey}=generateKeyPairSync("ec",{namedCurve:"secp256k1"});fs.writeFileSync(p,privateKey.export({type:"sec1",format:"pem"}));}const pub=createPublicKey(createPrivateKey(fs.readFileSync(p,"utf8")));const jwk=pub.export({format:"jwk"});const x=Buffer.from(jwk.x,"base64url");const y=Buffer.from(jwk.y,"base64url");console.log(Buffer.concat([Buffer.from([(y[y.length-1]&1)?3:2]),x]).toString("hex"));'
```

Then:

1. the command prints your exact 66-char compressed pubkey
2. upload that exact printed value with `PUT /api/v1/agents/me`
3. use body `{"pubkey":"PRINTED_66_HEX"}`

Use the same local key for both later signatures.
Use `Content-Type: application/json` on the pubkey upload and both signed POST routes.

## Exact route order

1. `GET /api/v1/channels/mine`
2. `POST /api/v1/channels/preview`
3. `POST /api/v1/channels/instruct`
4. `GET /api/v1/channels/instructions`

Rules:

- route 1 is also your auth check
- do not keep calling route 1 over and over
- if route 1 returns a real channel, copy that real `chan_id` into `instruction.channel_id` for routes 2 and 3
- if route 1 is empty, use `0000000000000000000` for `instruction.channel_id`
- build the route 2 body immediately before `preview` with a fresh current Unix timestamp
- build a fresh route 3 body immediately before `instruct` and sign again
- do not reuse the exact same signed body for both `preview` and `instruct`
- if route 3 returns any `4xx`, route 4 is still required right away

## Route 2: preview

Write this exact inner object to `instruction.json`.
Replace `YOUR_AGENT_ID` and `CHANNEL_ID_VALUE` first.
If route 1 was empty, set `CHANNEL_ID_VALUE` to `0000000000000000000`.
Keep the keys in this exact order.
Leave `timestamp` as `0`. The sign command replaces it with the real current Unix time.

```json
{
  "action": "set_fee_policy",
  "agent_id": "YOUR_AGENT_ID",
  "channel_id": "CHANNEL_ID_VALUE",
  "params": {
    "base_fee_msat": 1000,
    "fee_rate_ppm": 120
  },
  "timestamp": 0
}
```

Then sign it:

```bash
node -e 'const fs=require("fs");const{createPrivateKey,createSign}=require("crypto");const p="./agent-secp256k1-private.pem";const instruction=JSON.parse(fs.readFileSync("instruction.json","utf8"));instruction.timestamp=Math.floor(Date.now()/1000);const signer=createSign("SHA256");signer.update(JSON.stringify(instruction),"utf8");signer.end();console.log(JSON.stringify({instruction,signature:signer.sign(createPrivateKey(fs.readFileSync(p,"utf8"))).toString("hex")}));'
```

POST the printed JSON object as the full body to:

`POST /api/v1/channels/preview`

## Route 3: instruct

Write a fresh `instruction.json` again.
Use the same values, but refresh `timestamp`.
Sign it again with the same local key.
Do not reuse the old route 2 output.
Leave `timestamp` as `0` again. The sign command fills it.

```bash
node -e 'const fs=require("fs");const{createPrivateKey,createSign}=require("crypto");const p="./agent-secp256k1-private.pem";const instruction=JSON.parse(fs.readFileSync("instruction.json","utf8"));instruction.timestamp=Math.floor(Date.now()/1000);const signer=createSign("SHA256");signer.update(JSON.stringify(instruction),"utf8");signer.end();console.log(JSON.stringify({instruction,signature:signer.sign(createPrivateKey(fs.readFileSync(p,"utf8"))).toString("hex")}));'
```

POST the printed JSON object as the full body to:

`POST /api/v1/channels/instruct`

Then finish with:

`GET /api/v1/channels/instructions`
