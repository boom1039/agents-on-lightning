LIGHTNING NETWORK BASICS:

Channels: Bidirectional payment channels between Lightning nodes. Each channel has capacity, local balance, and remote balance.

Routing: Payments traverse multiple channels. Each forwarding hop can charge a routing fee.

Fees: Routing fees usually include a base fee and a proportional fee rate. Channel policy tools tune these economics.

Liquidity: Local balance is sats you can send outward. Remote balance is sats that can flow inward to you. Balanced or intentionally positioned channels can route better depending on demand.

Rebalancing: Moving liquidity inside existing channels. Rebalances can cost routing fees, so estimate first and execute only with real ownership, balance, and signatures.

COMMON FAILURE MODES:

1. "Signature invalid" - You signed the wrong data. Sign the exact instruction returned by the build tool using your local secp256k1 private key. Preview first when a preview tool exists.

2. "Channel not assigned to this agent" - You are trying to modify a channel you do not control. Use aol_get_channels_mine.

3. "Fee outside allowed range" - Your instruction violates channel constraints. Use aol_get_channels_mine and rebuild the instruction with allowed values.

4. "Cooldown active" - You changed fees too recently. Wait until the channel can be updated again.

5. "Insufficient balance" - Wallet funds, available capital, locked capital, pending deposits, and unpaid invoices are different states. Use wallet and capital balance tools before spending or signing.

6. "Invoice unpaid" - Payment is not complete. Check the quote or deposit status and stop honestly until the tool result says payment is complete.
